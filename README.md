# Minamo Score 

音声ファイルからピアノ譜を錬成する、ブラウザ完結型の魔術的アプリです。mp3、wav、またはマイク録音を突っ込むと、楽譜（表示・PDF・MusicXML）とMIDI（あなたの演奏のありのまま／綺麗に整えた版）を吐き出します。

**最大の安心ポイント：** 解析はすべてあなたの端末のブラウザ内で完結します。あなたの深夜のエモーショナルな弾き語り音源が、見知らぬサーバーに送信されることは絶対にありません。安心してください。

## 特徴

-   **高精度ピアノ採譜:** Kong大先生たちの「High-resolution Piano Transcription」を、ブラウザ上（onnxruntime-web）でゴリゴリ動かします。原典のPython実装と音符単位でピタリと一致するまで、テストで保証しています。
    
-   **汎用採譜:** 「ピアノじゃないんだけど？」というワガママにも対応。Spotifyのbasic-pitch（楽器非依存）が、あなたの謎のハミングも解読してくれます。なお、精度は・・
    
-   **CFP物証つき深掘り:** 採譜モデルとは別の証拠（スペクトルと一般化ケプストラムの積）を使って、検出閾値を深海まで下げつつ倍音の亡霊だけを華麗に蹴散らします。
    
-   **拍追跡:** 動的計画法の力で、味のあるテンポの揺らぎにAIが全力で追従。どんなに揺れても、16分音符主体の人間が読める綺麗な楽譜に仕立て上げます。
    
-   **爆速化への執念:** 80%ストライドの推論を事後確率のクロスフェードで縫い合わせます。さらに、環境が許せばWASMマルチスレッドが火を噴きます。
    
-   **履歴機能:** 変換結果と音声（30MBまで）を、IndexedDBにそっと保存します。現行エンジンでいつでも再変換して、過去の自分と向き合うことが可能です。
    

## 使い方

音声ファイルを画面にドロップして、「変換する」ボタンを押して祈るだけです。完了するとピアノロールと楽譜が現れます。

※ 注意：「音源の種類」をピアノにすると、初回のみ約48MBのAIモデルをダウンロードします。カップラーメンでも作って待っていてください（2回目以降はキャッシュが効くので速いです）。それ以外の楽器は汎用エンジンがサクッと処理します。

## 設置方法

ビルド済みの静的ファイルを、サーバーにポン置きするだけで動きます。

Plaintext

```
index.html  sw.js  manifest.webmanifest  styles/  dist/  models/
```

WASMマルチスレッドを呼び覚ますには少し面倒なヘッダ（COOP/COEP）が必要ですが、優秀なService Workerが勝手にヘッダを注入するため、サーバー側の設定すら不要です（初回アクセス時に1回だけ自動リロードが入りますが、バグではなく仕様です）。

もしサーバー側で設定できる権限をお持ちの恵まれた方は、`.htaccess`に以下を書くのが一番確実です。

Apache

```
Header set Cross-Origin-Opener-Policy "same-origin"
Header set Cross-Origin-Embedder-Policy "credentialless"
```

## 構成

| **パス** | **中身** |
| --- | --- |
| `src/00–13_*.ts` | UI・状態・音声入出力・音符整形など、表舞台の皆様 |
| `src/worker/` | 採譜エンジンの心臓部。純粋関数なのでNode.jsからも呼べる良い子たち |
| `models/` | AIの脳みそ（ONNXモデル） |

## 動作環境

Chrome、Edge、Firefox、Safariなどのモダンブラウザ。

※ マルチスレッドの恩恵を受けられるのは `crossOriginIsolated` な環境のみです。Safariは今のところシングルスレッドで汗水流して頑張ります。IEを使っている方は、まずタイムマシンで現代に帰ってきてください。
